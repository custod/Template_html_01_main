Общие сведения о разметке

Шаблон предназначен для статических веб-страниц (документации, скраббинга, изучения html и css, отработки навыков верстки, даже создания маленьких статических сайтов и т.д.)

Рекомендуется прочитать всё нижеследующее, прежде чем начинать работать с пустым шаблоном. А еще лучше держать данную страницу открытой в соседней вкладке браузера в качестве справки.

Шаблон сверстан с применением flexbox, семантических тегов, подсветки кода скриптом highlight.js Ивана Сагалаева, а также разных рюшечек (какие понравились автору).

В верстке шапки (на flexbox) количество параграфов внутри панели <nav> можно менять произвольно, с учетом ширины <nav>. Пока параграфы помещаются в ширине <nav>, об их расположении и отступах можно не беспокоиться. То же самое касается и футера.

Сайдбар и статья (основное содержание страницы) размечены в процентах, так что при необходимости изменить общую ширину шаблона или соотношение сайдбара (aside) и основного содержания (article) это несложно сделать, оперируя стилями для body, aside и article.

Шаблон (и данная демо-страница) сверстаны с такими значениями ширины:


body {
    width: 70%;
    height: auto;
    min-height: 100vh;
	/* etc */
}

aside {
	width: 33%;
	height: auto;
	position: sticky;
	top: 0;
	/* etc */
}

article {
	width: 65%;
	/* etc */
}

Можно заметить, что суммарная ширина aside и article меньше 100%, — так надежнее.

main служит лишь оберткой для aside и article, играя роль flexbox-родителя.

Структура шаблона

body = 70% от html
nav
main = 100% от body
aside = 33%
article = 65%
footer

Использование шаблона. Вставка содержимого

Содержимое каждой "главы" (секции) вставляется между тегами <section id="aN">...</section>

Для удобной ориентации в шаблоне между этими тегами есть закомментированные строки, вместо которых и нужно вставлять содержимое.
Вот как это выглядит:


                <section id="a2">
	<!--================= ✂ =================-->				
            <!--Вставлять содержимое здесь-->
	<!--================= ✂✂ =================-->	
		</section>

Обычный текст оформляется как параграфы, его свойства можно переопределить как глобально, в шаблоне 01_main_full.css, так и локально, только для текущей страницы в секции


        <head>
        ...............
           <style>
/*If need*/
            </style>
        </head>

Свойства задаются для селектора "article p":

article p {
    ..........
}

Аналогично можно переопределить стили других элементов.

Подсветка кода

осуществляется широко известным скриптом Ивана Сагалаева highlight.js. Простейший способ его использовать описан здесь. Вообще сопровождение у highlight.js очень хорошее: много документации, CDN библиотеки и т.д.

Пример для кода CSS:

article h2 {
    margin: 2rem auto 1.5rem;
    text-align: center;
    font: small-caps 1.8rem/2.5rem 'Istok Web', 'PT Sans Caption', sans-serif;
    color: #082B2B;
    letter-spacing: 2px;
}

Если подсветка не нужна, можно оформить блоки кода без подсветки, зато намного проще, при помощи селектора <pre class="code">. Результат выглядит так:

.code {
    width:98%;
    margin:.6rem auto;
    font:.9rem/1.2rem 'PT Mono',monospace;
    color:#18565e;
    border:2px solid #ccc7c1;
    padding:.2rem;
    background:#f4f9f9 content-box;
    overflow-x:auto
}

В этом случае для уменьшения шаблона 01_main.html в его конце можно удалить этот кусок:

<!--Highlight -->
            <script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.15.10/highlight.min.js"></script>
            <script>hljs.initHighlightingOnLoad();</script>
            <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.15.10/styles/color-brewer.min.css">
<!-- END Highlight -->

Выделить

всякие отдельные слова/выражения, сущности и прочее, на что требуется акцентировать внимание, можно несколькими способами, помимо традиционных bold, underline etc.

Важные сущности или пути можно выделить способом, содранным позаимствованным у Юлии Бухваловой, эксперта CSS и большого мастера всяких рюшечек :).

Кстати, у нее же подсмотрен прием оформления с помощью прямой вставки emoji

Оформляется селектором code с классом nohighlight (чтобы избежать конфликта с highlight.js):

code.nohighlight {
    /*от Бухваловой*/
    display: inline-block;
    background: #EDEDED none repeat scroll 0 0;
    color: #774D7F;
    font-family: 'PT Mono', monospace;
    font-size: 0.9rem;
    line-height: 1rem;
    padding: 0 5px;
    text-shadow: 2px 1px #fff;
    border-style: solid;
    border-color: #E3E9EA;
    border-width: 1px;
    border-radius: 3px;
}

Кроме того, можно применить тег kbd для выделения моноширинным шрифтом и неким аналогом полужирного шрифта при помощи <span class="select">.

Таблицы

Если на странице есть таблицы, в <head>...</head> нужно подключить еще один файл стилей:

<link rel="stylesheet" href="css/tables.css" media="all">

Этот файл (с некоторыми переделками) взят из фреймворка PureCSS.

Структура таблицы

Таблицы должны быть построены "правильно" :), то есть иметь такую стуктуру:

<table class="">
        <thead>
            <tr>
                <th>...</th>
                ............
                <th>...</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>...</td>
                ...........
                <td>...</td>
            </tr>
            .................
            <tr>
                <td>...</td>
                ...........
                <td>...</td>
            </tr>		
        </tbody>
    </table>

Проще всего формировать такую таблицу при помощи Emmet, который, конечно же, подключен плагином к вашему рабочему редактору или IDE.

Например, так:

table.main-table>(thead>tr>th*4)+tbody>tr*3>td*4

Позиционирование

Если требуется выравнивание таблиц по центру, а не по левому краю, в шаблон 01_main.html в секции <head><style>...</style></head> нужно просто добавить следующее:

.main-table {margin: 1rem auto;}

При этом будут центрированы все таблицы с классом main-table. Отдельную таблицу при необходимости можно перепозиционировать, определив ей стиль "на месте":

<table class="main-table main-table-striped" style="margin: 1rem auto;">

Вложенность в боковой панели "Содержание"

Структура боковой панели очень проста. Если же, как это нередко бывает, в ее пунктах потребуется создать подпункты, то можно пойти двумя путями.

Стандартный, или "классический" способ

состоит в том, чтобы нумеровать подпункты как потомки своих пунктов. Код при этом будет примерно такой:

<aside>
                    <header>
                        <h1>Содержание</h1>
                    </header>
                    <ul>
            <li><a href="#a1" target="_self">section 1</a></li>
            <li><a href="#a2" target="_self">section 2</a>
                <ul>
                    <li><a href="#a2-1" target="_self">section 2-1</a>
                    <li><a href="#a2-2" target="_self">section 2-2</a>
                </ul></li>
            <li><a href="#a3" target="_self">section 3</a></li>
            .........................................................
                    </ul>
        </aside>

На мой взгляд, с таким кодом неудобно работать, поэтому рекомендуется использовать

"Сквозной" порядок нумерации

При этом все пункты и подпункты нумеруются строго по порядку, примерно так:

	<aside>
                <header>
                    <h1>Содержание</h1>
                </header>
                <ul>
        <li><a href="#a1" target="_self">section 1</a></li>
        <li><a href="#a2" target="_self">section 2</a>
            <ul>
                <li><a href="#a3" target="_self">section 2-1</a></li>
                <li><a href="#a4" target="_self">section 2-2</a></li>
            </ul></li>
        <li><a href="#a5" target="_self">section 5</a></li>
        ............................................................
                </ul>
    </aside>

Такой порядок, имхо, понятнее и проще, если содержание заранее ясно и меняться не будет. Если же возникает нужда вставить в уже существующую структуру, где-то в ее середке, дополнительный подпункт, то вполне подойдет и "классический" способ, иначе придется перенумеровывать все секции.

Адаптивность

Как показало беглое тестирование, шаблон неплохо масштабируется вплоть до размеров планшетов. Но верстку нельзя назвать по-настоящему адаптивной, поскольку она рассчитана на работу за компьютером или ноутбуком. Телефоны/смартфоны не предназначены для чтения документации и тем более для отработки веб-дизайна.
